– client:willSubmitEsign:fromViewController:completionHandler:

If implemented will trigger notify when user going to submit a e-sign

- (void)client:(MEPClient *)client willSubmitEsign:(NSString *__nullable)payload fromViewController:(UIViewController *)viewController completionHandler:(void ( ^ _Nullable ) ( BOOL pass , NSString *__nullable jwt ))completionHandler

Parameters

client

The mep sdk client
payload

The payload of this e-sign submit action
viewController

From which view controller this action happens
completionHandler

A block object which should be called when you decide user can submit this e-sign or not.

Discussion

This completionHandler block is required to be called immediately after your process done. First parameters is boolean, to indicate if SDK should submit esign, for example, per 2FA case, once completed the authentication, you should call pass ‘YES’ to first parameter. Second parameter is optional JWT,to provide security check per esign .
